.What is the difference between var, let, and const?
A:
	var:
		- Function-scoped : Accessible within the function it's declared in.

		- Hosting is possible and they are initialized with undefined until the line of declaration is executed.
		- var allows both reassignment and redeclaration within the same scope.

	let:
		- let declarations are block-scoped.
		- let declarations are hoisted to the top of their block but are not initialized
		- let allows reassignment but does not allow redeclaration within the same block scope.
	
	const:
		- const declarations are also block-scoped, similar to let.
		- const declarations are hoisted but not initialized
		- const variables must be initialized at the time of declaration and cannot be reassigned afterward.


.What is Hoisting in JavaScript?
A:
	Hoisting is JavaScript's default behavior of moving declarations (not initializations) to the top of the current scope.

	Ex:
		sayHello(); // Output: Hello!

		function sayHello() {
 		console.log("Hello!");
		}

  	-Compares values after converting them to the same type.

   	-It's more flexible but can lead to unexpected results.


	 === (Strict Equality):
	
	- Compares both value and type.

.What is the scope of a variable in JavaScript?
A:

| Scope Type     | Where It's Accessible                      | Declared With         |
| -------------- | ------------------------------------------ | --------------------- |
| Global Scope   | Anywhere in the file (or window)           | `var`, `let`, `const` |
| Function Scope | Inside the function where declared         | `var`, `let`, `const` |
| Block Scope    | Inside the `{}` block where declared       | `let`, `const`        |
| Lexical Scope  | Inner functions accessing parent variables | All declarations      |



.Explain event bubbling and event capturing.
A:
	Event Capturing : The event starts from the root of the DOM tree and travels downward to the target element( document -> parent->target).
	
	Event Bubbling : After reaching the target element, the event bubbles back up to the top (from target → parents → document).



.What is the event loop in JavaScript?
A: 
	The event loop in JavaScript is a core concept that explains how JavaScript handles asynchronous operations — like setTimeout, fetch, promises, and event handlers — while still running in a single thread.


.What are promises, and how do they work?
A:

	A Promise is a built-in JavaScript object that represents the eventual result (or failure) of an asynchronous operation.

	| State       | Description                      |
	| ----------- | -------------------------------- |
	| `pending`   | Initial state, still running     |
	| `fulfilled` | Operation completed successfully |
	| `rejected`  | Operation failed with an error   |



.Explain async/await.
A:
	async Keyword:

	- Used before a function to declare it as asynchronous.

	- Automatically makes the function return a Promise.

	
	await Keyword:

	- Can only be used inside an async function.

	- Pauses execution until the Promise is resolved or rejected.

	- Makes asynchronous code behave like synchronous code.


.What is a callback function?
A:
	A callback is a function passed as an argument to another function. 
	This technique allows a function to call another function.
	A callback function can run after another function has finished.

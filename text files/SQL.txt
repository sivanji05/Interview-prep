5)Explain  types of SQL commands in detail?	
A:
       ◦Data Definition Language: -> Defines and modifies the database structure.

			•CREATE	:-Creates a new database, table, index, or view.   -> Example: CREATE TABLE students (id INT, name VARCHAR(50));

			•ALTER	:- Modifies an existing database object, such as adding or dropping columns in a table.   ->Example: ALTER TABLE students ADD COLUMN age INT;

			•RENAME	:- Renames a table or a column.     ->Example: RENAME TABLE old_table TO new_table;

			•TRUNCATE :- Removes all rows from a table, but the table structure remains intact.     -> Example: TRUNCATE TABLE students;

			•Drop	:- Deletes the entire table or database structure.    -> Example:  DROP TABLE employees;
	
	◦Data Manipulation  Language:  ->Manages and manipulates data.

			•INSERT	:- Adds new records to a table.    -> Example: INSERT INTO students (id, name) VALUES (1, 'John');

			•UPDATE	:- Modifies existing records in a table.    -> Example: UPDATE students SET name = 'Jane' WHERE id = 1;

			•DELETE	:- Removes records from a table based on a condition.   -> Example: DELETE FROM students WHERE id = 1;

	◦Data Control Language : Controls access to the database.

			•GRANT	:- Gives specific privileges to a user.   -> Example: GRANT SELECT, INSERT ON database_name.* TO 'username'@'localhost';

			•REVOKE	:- Removes specific privileges from a user.    -> Example: REVOKE SELECT ON database_name.* FROM 'username'@'localhost';
	
	◦Data Query Language : Retrieves data using queries.
			
			•SELECT	:- Retrieves data from one or more tables. Example: SELECT * FROM students;

	◦Transaction Control Language :Manages transactions to ensure data consistency.

			•COMMIT	:- Saves the changes made by the transaction permanently.   -> Example: COMMIT;
			•ROLLBACK  :- Reverts the changes made by the current transaction.  -> Example: ROLLBACK;


6) What is the WHERE,ORDER BY,GROUP BY clause used for?
A: 
	- WHERE clause It filters records based on specified conditions.

	- ORDER BY clause : It sorts the result set in ascending (ASC) or descending (DESC) order.

	- GROUP BY clause : It groups rows with the same values into summary rows.



7)what are the Constraints ?
A: 
	The following constraints are commonly used in SQL:

	-NOT NULL  	:- Ensures that a column cannot have a NULL value

	-UNIQUE  	:- Ensures that all values in a column are different

	-PRIMARY KEY 	:- A combination of a NOT NULL and UNIQUE. Uniquely identifies each row in a table

	-FOREIGN KEY 	:- Prevents actions that would destroy links between tables

	-CHECK		:- Ensures that the values in a column satisfies a specific condition

	-DEFAULT 	:- Sets a default value for a column if no value is specified
   
	-CREATE INDEX 	:- Used to create and retrieve data from the database very quickly

8)Differences Between DROP and TRUNCATE?
A: 
	- DROP: Deletes the entire table or database structure.

	- TRUNCATE: Deletes all rows from a table but keeps the table structure intact.

12)Write the Syntaxes for joins.
A:
	-INNER JOIN: select column_name From table1 INNER JOIN table2 on table1.column_name = table2.column_name;
	-OUTER JOIN: select column_name From table1 OUTER JOIN table2 on table1.column_name = table2.column_name;
	-LEFT JOIN: select column_name From table1 LEFT JOIN table2 on table1.column_name = table2.column_name;
	-RIGHT JOIN: select column_name From table1 RIGHT JOIN table2 on table1.column_name = table2.column_name;
	-FULL JOIN: select column_name From table1 INNER JOIN table2 on table1.column_name = table2.column_name ;
